Part I: Write JavaScript code to validate Input

Please validate the following input element using the rules mentioned below each one. An input element is just like any other element in HTML just that it allows one to enter text on one line. This is what is typically used to accept various such inputs on forms used on a webpage

You will notice that next to each of the submit button is associated with a function. You are supposed to write the function. I have added empty function bodies at the bottom in the <script> block

Next to each of the Submit button, you will see written in gray a label that says that the input is currently "unverified". Once you verify the input, you should replace this text with the text verified, or error. Use the exact strings and style as in this .html file for the sake of consistenty and automated testing

Integer: unverified
(10 points)

  • Should be an Integer between 1 and 100
  • Should not accept a string or any other value that is not an Integer

Full Name: unverified
(10 points)

  • Should only contain two or three words
  • If it contains three words, then the first one should either me "Prof.", "Mr.", "Dr." or the last word should be "Jr.", "Sr.", or "Ph.D."
  • The two words should be capitalized
  • The words should not contain any numbers, and if it contains a punctuation it should only be a "-" in the last name
Email: unverified(10 points)

  • Should end with one of the allowable Top Level Domains (TLD). Hint: You can find a complete list on Wikipedia
  • It should contain a @
  • The username (part before the '@') should only contain alphanumeric characters and a "-" or a ".". No other characters should be allowed
  • The username should be at most 20 characters long

Password: unverified
(10 points)

The password should ...

  • contain at least one uppercase letter
  • contain at least one lowercase letter
  • contain at least one punctuation
  • contain at least one number
  • be at least 8 characters and at most 50 characters long
  • cannot contain spaces
  • cannot contain any repeated sub-strings of any length
  • cannot contain a substring or any space-separated word token occuring in the firstname, lastname, email, date of birth, month of birth, or year of birth
  • Extra points: Disallow pasting in the password field

Birth Date: unverified
(10 points)

  • Should be a valid date as accepted without any error by the Javascript Date module
  • The date should be at least 10 years before the current date
  • Once the date is accepted, and when the field loses focus you should replace the content of the input field with the following stadardized version YYYY/MM/DD.

State: unverified
(10 points)

  • Should be a two character abbreviation or a capitalized fullname. You should accept lowercase two characters if there is an exact match after converting them to uppercase. If they enter the capitalized name, then it should be entered in the correct case.
  • It should be validated by the list of state names and abbreviations in the file us-states.json file
  • When this input field looses focus, you should replace the contents with the normalized two character state in all uppercase

Zipcode: unverified
(10 points)

  • Should be five digit code optionally followed by a dash and another four digits as the allowed by the plus-four or ZIP+4 postal standard
  • The first five digit zipcode should be a valid zipcode. You will find a valid-zipcodes.json file in your repository. The content of the file is a JSON object. You can find out how to read a JSON object in Javascript at the following MDN webpage
  • Extra points: Ensure that the first digit of the zipcode belongs to the specified state by checking the rule mentioned in the plain text file <first-digits-of-us-state-zipcodes.txt
  • Extra points: Add the first digit rules to the us-states.json file and use that in the code
